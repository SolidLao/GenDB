================================================================================
QUERY LOGIC VERIFICATION
================================================================================

Input Query:
  SELECT l_orderkey, SUM(l_extendedprice * (1 - l_discount)) AS revenue,
         o_orderdate, o_shippriority
  FROM customer, orders, lineitem
  WHERE c_mktsegment = 'BUILDING'
    AND c_custkey = o_custkey
    AND l_orderkey = o_orderkey
    AND o_orderdate < DATE '1995-03-15'
    AND l_shipdate > DATE '1995-03-15'
  GROUP BY l_orderkey, o_orderdate, o_shippriority
  ORDER BY revenue DESC, o_orderdate ASC
  LIMIT 10;

Implementation:
  1. Load customer, filter by c_mktsegment=0 (BUILDING)
     → 1.5M → 300K customers
  2. Load orders, filter by o_custkey in {filtered customers}
     → 15M → 1.46M orders
  3. Load lineitem, filter by l_shipdate > 9204 and l_orderkey in {filtered orders}
     → 60M rows scanned, matches found
  4. For each matching lineitem:
     - Compute revenue = l_extendedprice * (100 - l_discount)
     - Aggregate by (l_orderkey, o_orderdate, o_shippriority)
     → 114K unique groups
  5. Sort by revenue DESC, o_orderdate ASC
  6. Take first 10 rows
  7. Output as CSV

Result: 10 rows, byte-for-byte match with ground truth ✓

PERFORMANCE OPTIMIZATION
================================================================================

Techniques Used:
  1. Early filtering (customer → orders → lineitem)
  2. Hash tables for O(1) lookup during joins
  3. Single-pass aggregation using unordered_map
  4. Vectorizable predicates (integer comparisons)
  5. mmap for zero-copy column loading
  6. C++17 compilation with -O3 -march=native

Timing Breakdown (4766 ms total):
  - Data loading: ~14 ms (3 tables via mmap)
  - Filtering: ~1560 ms (customer, orders, lineitem predicates)
  - Joining & aggregation: ~3100 ms (hash map construction + lineitem scan)
  - Sorting: ~8 ms (114K groups)
  - Output: ~0.2 ms (10 rows to CSV)

Optimization Notes:
  - Initial naive O(n*m) orderkey lookup fixed to O(n) by iterating through orders once

  - Standalone hash struct avoids std namespace specialization pitfalls
