WHY THIS WORKS:
---------------

1. Cache Locality: Open-addressing stores keys and values contiguously,
   improving spatial locality compared to chained hashing which scatters
   entries across heap allocations.

2. Reduced Allocations: std::unordered_map allocates a node per entry.
   Open-addressing preallocates a flat array, eliminating per-entry overhead.

3. Better Memory Layout: Linear probing keeps related data close together,
   reducing cache misses during lookups in tight loops.

4. Fast Hash Function: MurmurHash3 finalizer provides excellent avalanche
   properties with minimal computation cost.

5. Power-of-2 Sizing: Uses bitwise AND instead of modulo for fast addressing.
